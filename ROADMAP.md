# Roadmap

This roadmap tracks the next layers of the LLM Wiki project.

## v0.1 — Published foundation

Completed:
- LM Studio local backend
- Dockerized watchdog daemon
- OpenAI-compatible LLM client
- Inbox ingest pipeline
- Startup backlog scan
- GitHub publication baseline

## v0.2 — Knowledge maintenance

Planned:
- Lint pass for wiki health checks
- Detection of orphan pages and missing links
- Better metadata consistency checks
- Safer handling of malformed model outputs

## v0.3 — Knowledge graph enrichment

Planned:
- Automatic interlinking between nodes
- Stronger entity and concept normalization
- Contradiction detection across accumulated notes
- Improved relationship extraction for concept graph growth

## v0.4 — Better authoring and review loop

Planned:
- Richer review workflow for generated nodes
- Better source provenance and update tracking
- Improved visual assets and example datasets
- More polished release process for public iterations

## Maintenance Rule

As the project evolves, roadmap milestones and publication-facing docs should be updated together when relevant, especially:

- `README.md`
- `README.zh-CN.md`
- `CONTRIBUTING.md`
- GitHub issue templates
- PR template

The goal is to keep the repository state, collaboration guidance, and public presentation aligned.
