# LLM Wiki 初始化技术纪实

今天我们从极简 CLAUDE.md 开始，选择了“指针模式记忆”，避免把全部规范塞进上下文，而是只保留指向 docs 的入口。这样做的原因是上下文有限，架构知识应当按需加载，而不是每次都污染会话。

随后，我们决定将知识库从“手动脚本”升级到“容器化监听”。原因很直接：如果每次摄入都要求人手工敲命令，知识库迟早会因为维护摩擦而失去活性。只有让监听常驻、摄入自动发生，知识才会真正积累。

在基础设施上，我们选了 OrbStack + Docker Compose。因为它在 macOS 上的 bind mount 性能更好，适合 watchdog 这种依赖文件系统事件的常驻监听场景。Obsidian 负责可视化浏览，容器负责后台执行，二者通过挂载目录实时同步。

最后，我们打通了 ingest pipeline：文件进入 inbox，daemon 触发 loader，LLM 提取摘要、关键主张、实体与概念，再生成标准化 Markdown 笔记并归档原文。这意味着知识库已经从静态文件夹变成了一个会自动生长的系统。
